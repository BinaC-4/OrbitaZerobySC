    Internet ✶⋆.˚

    O que é internet?

    São cabos submarinos conectados a servidores localizados em datacenters.

    A internet é uma rede global de computadores interconectados, que permite a troca de dados e informações entre dispositivos em qua1lquer lugar do mundo, por meio de protocolos de comunicação padronizados, principalmente o TCP/IP.

    Origem da Internet

    A internet foi criada a partir da ARPANET, desenvolvida nos Estados Unidos no final da década de 1960. Sua motivação original era militar: garantir a continuidade da comunicação e o acesso a informações estratégicas mesmo em caso de ataques, como um ataque nuclear. Hoje, seu uso é amplamente diferente, voltado para fins civis e comerciais.

    A internet como conhecemos hoje só surgiu nos anos 1980-1990.

    ➜ ARPANET (Advanced Research Projects Agency Network): foi a primeira rede de computadores de comutação de pacotes da história e é considerada o embrião da internet.

    Termos mais comuns

    ➜ Datacenter (Centro de Dados): é uma instalação física projetada para abrigar sistemas computacionais críticos, como:

    • Servidores (computadores de alto desempenho);
    • Dispositivos de armazenamento (HDs, SSDs);
    • Equipamentos de rede (roteadores, switches, firewalls);
    • Fontes de energia redundante e sistemas de resfriamento;
    • Função principal: Armazenar, processar e distribuir dados;
    • Um datacenter é o coração físico da internet. É onde ficam os computadores que armazenam sites, aplicativos, bancos de dados, e-mail, vídeos e toda a informação digital que circula globalmente;
    • Exemplos de empresas que operam datacenters: Google, Amazon (AWS), Microsoft (Azure), Meta, IBM, Oracle.

    ➜ IP (Internet Protocol / Protocolo de internet): é o protocolo que permite localizar, identificar e enviar dados entre dispositivos na internet. É o sistema de endereçamento da rede mundial.

    • Qualquer máquina conectada a uma rede possui um endereço de IP, uma série de números única que serve para identificar o dispositivo na rede, como o nosso CPF! Ele tem essa cara: 74.125.19.147
    • Função principal: Permitir que dados sejam enviados de um dispositivo a outro (computadores, servidores, roteadores, celulares) por meio de endereços únicos chamados endereços IP;
    • O IP funciona junto com outros protocolos, especialmente:
      • TCP (Transmission Control Protocol) → Garante que os dados cheguem completos e na ordem correta.
        • Mais seguro - usado em mensagens, por exemplo. Com garantia de que sua mensagem será enviada
      • DNS (Domain Name System) → Traduz nomes como google.com em endereços IP.
        • Mais rápido - usado para ligações e chamadas, por exemplo. Sem garantia de que os dados serão entregues
    • É uma das bases técnicas da internet.

    O que é um website?

    Um website é um conjunto de arquivos escritos em linguagens como HTML, CSS e JavaScript, organizados em um servidor, que o navegador interpreta e exibe ao usuário.

    ⎆ Arquivos codificados que são interpretados por navegadores

    Linguagens que constituem esses arquivos

    • HTML - Define a estrutura de elementos (textos, títulos, links)
    • CSS - define a aparência (cores, fontes, layout)
    • JavaScript - define os comportamentos e interações (menus, formulários, animações)
    ❝ Também vale diferenciar a Internet da Web! Apesar de muitas vezes usarmos como sinônimos, é importante saber que a Internet é a infraestrutura, a rede de computadores e dispositivos que trocam informações, e a Web é um serviço construído utilizando essa infraestrutura. Via Internet, você utiliza a web por meio dos navegadores, responsáveis pela requisição e apresentação das informações que nos são servidas/devolvidas pelos servidores. Por exemplo, Torrent, troca de mensagens simultâneas e aplicativos são outros serviços que utilizam a infraestrutura da internet e não necessariamente a web! ❞ - Progra{M}aria

    Simplificando

    Quando fazemos pesquisas em navegadores, enviamos e-mails ou acessamos receitas, estamos solicitando dados que estão armazenados em servidores remotos. Esses servidores nos enviam arquivos compostos por códigos (HTML, CSS, JavaScript, etc.), que são interpretados pelos navegadores e exibidos como páginas, textos, imagens ou vídeos.

    Antes de chegar até nós, esses dados trafegam por uma rede de servidores e roteadores — passando por diversos pontos até atingir o nosso dispositivo.

    Onde esses dados ficam?

    Em servidores localizados em datacenters ao redor do mundo. Sites, e-mails, vídeos, buscas, tudo é armazenado nesses servidores.

    O que acontece quando você pesquisa algo?

    1. O navegador envia uma requisição para o servidor do google, por exemplo;
    2. Esse pedido é roteado pela internet - passa por vários dispositivos (roteadores, serviços, cabos);
    3. Quando chega ao servidor certo, ele responde com arquivos contendo o conteúdo da busca;
    4. Esses arquivos, compostos por códigos de programacão, são interpretados pelo navegador, que exibe então conteúdo, o resultado visual.

    HTML ✶⋆.˚

    Introdução ao HTML

    O que é?

    HTML – HyperText Markup Language (Linguagem de Marcação de Hipertexto)

    HTML é a linguagem usada para estruturar o conteúdo de páginas da web. Ela define a hierarquia e a organização dos elementos, como textos, títulos, listas, links, imagens, vídeos e tabelas. É o esqueleto da página.

    Importante: HTML não é uma linguagem de programação. Trata-se de uma linguagem de marcação, usada apenas para organizar e apresentar o conteúdo — sem lógica, condições ou interações dinâmicas.

    Como funciona a estrutura de um documento HTML?

    Um documento HTML é organizado em uma estrutura hierárquica, que define como os elementos se relacionam e são apresentados na página. A estrutura básica de um documento HTML inclui as seguintes partes:

    1. Declaração do tipo de documento: <!DOCTYPE html> - informa ao navegador que este é um documento HTML.
    2. Elemento raiz: <html> - envolve todo o conteúdo da página.
    3. Elemento head: <head> - contém metadados, como título, links para estilos e scripts.
    4. Elemento body: <body> - contém o conteúdo visível da página, como textos, imagens e links.

    Exemplo de estrutura básica:

        <!DOCTYPE html>
        <html>
            <head>
                <title>Título da Página</title>
                <link rel="stylesheet" href="styles.css">
            </head>
            <body>
                <h1>Bem-vindo ao meu site!</h1>
                <p>Este é um parágrafo de exemplo.</p>
                <a href="https://www.exemplo.com">Clique aqui</a>
            </body>
        </html>

    Essa estrutura é a base de qualquer página HTML. A partir dela, você pode adicionar mais elementos e personalizar o conteúdo.

    Tags HTML

    O que são?

    As tags HTML são elementos fundamentais da linguagem. Elas funcionam como “etiquetas” que indicam o papel de cada parte do conteúdo em uma página — títulos, parágrafos, listas, links, imagens, tabelas, entre outros.

    Cada tag é escrita entre sinais de “menor que” e “maior que” (< >). A maioria das tags possui abertura e fechamento:

    <p>Texto desejado</p>

    No exemplo acima:

    • <p> abre um parágrafo;
    • </p> fecha o parágrafo.

    A barra / dentro da tag indica o fim daquele elemento.

    Nem todas as tags exigem fechamento. Algumas são chamadas de tags autoencerradas, como <img> (imagem) e <br> (quebra de linha). Essas não possuem conteúdo interno e são usadas sozinhas.

    HTML é composto quase totalmente por tags, por isso é essencial conhecê-las bem. Para uma lista completa, consulte a documentação oficial:

    • Lista completa de elementos HTML na MDN

    Principais Tags e suas funções

    Algumas das principais tags HTML que você deve conhecer são:

    • <h1> a <h6>: Títulos, do mais importante (h1) ao menos importante (h6).
    • <p>: Parágrafo.
    • <a>: Link (âncora).
    • <img>: Imagem.
    • <ul>, <ol>, <li>: Listas não ordenadas (ul), ordenadas (ol) e itens de lista (li).
    • <div>: Divisão ou seção genérica.
    • <span>: Span, usado para estilizar partes específicas do texto.
    • <br>: Quebra de linha.
    • <table>, <tr>, <td>, <th>: Tabela, linha de tabela, célula de tabela e cabeçalho de tabela.

    Essas tags são fundamentais para estruturar o conteúdo de uma página HTML. Você pode combiná-las e aninhá-las para criar layouts complexos e organizar informações de forma clara.

    Elementos sêmanticos

    O que são e qual sua importância

    Elementos semânticos são tags HTML que descrevem claramente o tipo de conteúdo que envolvem, indo além da simples estruturação visual. Eles indicam o propósito e a função do conteúdo para navegadores, leitores de tela, motores de busca e outros agentes automatizados.

    A importância dos elementos semânticos está em diversos aspectos:

    • Acessibilidade: leitores de tela e dispositivos assistivos interpretam melhor o conteúdo, proporcionando uma navegação mais inclusiva para pessoas com deficiência;
    • SEO (Otimização para motores de busca): motores de busca utilizam a semântica para entender a hierarquia e relevância das informações, melhorando o posicionamento nos resultados;
    • Manutenção e leitura do código: torna o código mais organizado, legível e fácil de manter por outros desenvolvedores;
    • Compatibilidade futura: garante que o site siga padrões web modernos, facilitando a adaptação a novas tecnologias e navegadores.

    Portanto, o uso correto dos elementos semânticos é fundamental para a criação de páginas web eficientes, acessíveis e otimizadas.

    Exemplos principais:

    • <header>: Cabeçalho da página ou seção.
    • <nav>: Navegação, links para outras páginas ou seções.
    • <main>: Conteúdo principal da página.
    • <article>: Artigo independente, como uma postagem de blog.
    • <section>: Seção genérica de conteúdo.
    • <aside>: Conteúdo relacionado, como uma barra lateral.
    • <footer>: Rodapé da página ou seção.

    Atributos HTML

    O que são atributos HTML?

    Atributos HTML são informações adicionais que podem ser adicionadas às tags para modificar seu comportamento ou fornecer informações extras. Eles são escritos dentro da tag de abertura, após o nome da tag.

    Exemplo:

        <a href="https://www.exemplo.com" target="_blank">Clique aqui</a>

    No exemplo acima, a tag <a> (link) possui dois atributos:

    • href: Define o endereço do link;
    • target: Especifica onde o link será aberto (neste caso, em uma nova aba).

    Quais são os principais atributos?

    Alguns dos principais atributos HTML que você deve conhecer são:

    • id: Identificador único para o elemento;
    • class: Classe CSS para estilização;
    • style: Estilo inline para o elemento;
    • src: Fonte de uma imagem ou script;
    • alt: Texto alternativo para imagens;
    • href: Endereço de um link;
    • target: Especifica onde o link será aberto (ex: _blank para abrir em nova aba);
    • title: Texto descritivo exibido ao passar o mouse sobre o elemento;
    • lang: Define o idioma do conteúdo;

    Esses atributos são amplamente utilizados para estilização, acessibilidade e funcionalidade dos elementos HTML. Você pode combiná-los conforme necessário para atender às necessidades do seu projeto.

    Mais detalhes

    Link:

    • Tag usada para vincular um recurso externo ao HTML
    • Ela não tem fechamento (), pois é uma tag autossuficiente (auto-fechada)
    • Não serve só para CSS - também é usada para ícones, fontes externas, etc.

    Atributo rel:

    • Significa "relationship” - relação
    • Diz qual é a relação entre o HTML e o arquivo que está sendo linkado.
    • No caso: rel="stylesheet” → indica que o arquivo externo é uma folha de estilo CSS

    Atributo href:

    • Significa "hyperlink reference”
    • Aponta onde está localizado o arquivo CSS que será usado.
    • Exemplo: href:"style.css” → o navegador vai buscar um arquivo chamado style.css na mesma pasta onde está o arquivo HTML

    Stylesheet:

    • Significa "folha de estilos”
    • Diz ao navegador: “o arquivo que estou linkndo é uma folha de estilo css". Ou seja, o navegador entende que o conteúdo desse arquivo deve ser usado para estilizar a página HTML (cores, fontes, tamanhos, etc)
    • IMPORTANTE! Se você trocar stylesheet por qualquer outra palavra, o navegador não vai aplicar o CSS, porque ele não vai reconhecer o tipo de relação.

    Atributo target:

    • Significa "alvo”
    • Indica onde o link deve ser aberto
    • Exemplo: target="_blank" → abre o link em uma nova aba do navegador

    Simplificando:

    • link : Tag usada para vincular um recurso externo ao HTML
    • stylesheet : Arquivo de estilo CSS
    • rel : Define a relação entre o HTML e o arquivo externo
    • href : Caminho até o arquivo CSS
    • target : Onde o link será aberto

    Boas práticas e de escrita em HTML

    Escrever um código HTML limpo e organizado é essencial para facilitar a manutenção e a compreensão do projeto. Além disso, também garante que seu site seja acessível e seja bem interpretado pelos navegadores e mecanismos de busca.

    Algumas boas práticas incluem:

    • Utilizar tags semânticas sempre que possível, para melhorar a acessibilidade e a compreensão do conteúdo;
    • Organizar o código com indentação adequada, para facilitar a leitura;
    • Fechar as tags corretamente, para evitar problemas de renderização;
    • Organização por seções
    • Utilizar comentários para explicar partes do código, se necessário;
    • Evitar o uso excessivo de tags inline, preferindo classes e IDs para estilização;
    • Utilizar atributos alt em imagens, para melhorar a acessibilidade;

    Organização do código

    Um código bem organizado facilita a leitura, manutenção e colaboração com outras pessoas. Estruture os elementos em uma hierarquia clara, agrupe conteúdos relacionados e evite duplicações desnecessárias. Cada seção da página deve ter uma função específica e delimitada, com uso adequado de <div>, <section>, <article>, entre outros.

    Uso correto das tags semânticas

    Evite usar <div> ou <span> para tudo. Prefira elementos semânticos como <header>, <nav>, <main>, <section>, <article>, <aside> e <footer>. Eles informam ao navegador e a outros agentes qual o papel de cada conteúdo, melhorando a acessibilidade, o SEO e a estrutura do site.

    Indentação

    A indentação correta — com dois ou quatro espaços por nível — ajuda a visualizar a hierarquia dos elementos. Todo conteúdo filho deve ser recuado em relação ao elemento pai. Isso torna o código mais legível e reduz a chance de erros de fechamento de tags.

    Exemplo:

    <ul>
  <li>Item 1</li>
  <li>Item 2</li>
</ul>

    Observação: Um código bem escrito e organizado faz com que o SEO (Search Engine Optimization) funcione melhor, pois os mecanismos de busca conseguem entender melhor o conteúdo da página. E, caso seu código tenha erros, o navegador pode não exibir a página corretamente, ou até mesmo lhe penalizar nos resultados de busca.

    CSS ✶⋆.˚

    Introdução ao CSS

    O que é CSS?

    CSS – Cascading Style Sheets (Folhas de Estilo em Cascata)

    CSS é a linguagem usada para estilizar e formatar o conteúdo de páginas HTML. Ela define como os elementos devem ser exibidos, incluindo cores, fontes, tamanhos, espaçamentos e layout. É responsável pela aparência visual da página.

    Importante: CSS também não é uma linguagem de programação. Trata-se de uma linguagem de estilo, usada apenas para definir a apresentação visual do conteúdo — sem lógica ou interações dinâmicas.

    Como funciona?

    O CSS funciona aplicando regras de estilo aos elementos HTML. Essas regras são compostas por seletores e declarações, que definem quais elementos serão estilizados e como.

    Exemplo de regra CSS:

            h1 {
            color: blue;
            font-size: 24px;
        }

    No exemplo acima:

    • h1 é o seletor, que indica que a regra se aplica a todos os elementos <h1>;
    • color: blue; é uma declaração que define a cor do texto como azul;
    • font-size: 24px; é outra declaração que define o tamanho da fonte como 24 pixels.

    As regras CSS podem ser aplicadas a qualquer elemento HTML, permitindo personalizar a aparência de textos, imagens, links, tabelas e muito mais.

    As propriedades mais comuns do CSS incluem:

    • color: Cor do texto;
    • background-color: Cor de fundo;
    • font-size: Tamanho da fonte;
    • font-family: Família da fonte;
    • margin: Margem externa;
    • padding: Preenchimento interno;
    • border: Borda do elemento;
    • width e height: Largura e altura do elemento;

    Como incluir em um documento HTML?

    Adicionar CSS a um documento HTML pode ser feito de três maneiras principais:

    1. Inline: Adicionando estilos diretamente na tag HTML usando o atributo style.
      2.         <h1 style="color: blue; font-size: 24px;">Título</h1>
    2. Internal: Incluindo um bloco de estilos dentro da tag <head> do HTML.
      3.         <head>
            <style>
                h1 {
                    color: blue;                    
                    font-size: 24px;
                }
            </style>
        </head>
    3. External: Criando um arquivo CSS separado e vinculando-o ao HTML.
      4.         <head>
            <link rel="stylesheet" href="styles.css">
        </head>

    A forma mais recomendada é a externa, pois permite separar o conteúdo (HTML) da apresentação (CSS), facilitando a manutenção e reutilização dos estilos em várias páginas.

    Seletores

    Seletores CSS são usados para selecionar quais elementos HTML serão estilizados. Existem vários tipos de seletores, cada um com sua finalidade:

    • Seletores de tipo: Selecionam todos os elementos de um determinado tipo (ex: h1, p, div).
    • Seletores de classe: Selecionam elementos com uma classe específica (ex: .minha-classe).
    • Seletores de ID: Selecionam um elemento com um ID específico (ex: #meu-id).
    • Seletores de atributo: Selecionam elementos com um atributo específico (ex: [type="text"]).
    • Seletores combinados: Combinam diferentes seletores para maior precisão (ex: div.minha-classe, h1#meu-id).

    Exemplo de uso de seletores:

        p {
        color: red;
    }
    .minha-classe {
        font-size: 20px;
    }
    #meu-id {
        background-color: yellow;
    }

    Esses seletores são usados no HTML e, mais tarde, especificados no CSS para estilização.

    Importante: ID deve ser único em uma página, enquanto classes podem ser reutilizadas em vários elementos.

    Unidades de medida

    CSS utiliza diferentes unidades de medida para definir tamanhos, distâncias e proporções. As mais comuns são:

    • px: Pixels, unidade fixa baseada na resolução da tela;
    • em: Unidade relativa ao tamanho da fonte do elemento pai;
    • rem: Unidade relativa ao tamanho da fonte do elemento raiz (html);
    • %: Porcentagem, relativa ao tamanho do elemento pai;
    • vh: Viewport height, 1% da altura da janela de visualização;
    • vw: Viewport width, 1% da largura da janela de visualização.

    Exemplo de uso de unidades de medida:

        h1 {
        font-size: 2em;
        margin: 10px;
    }
    .container {
        width: 80%;
        height: 50vh;
    }

    Cores

    As cores no CSS podem ser definidas de várias maneiras:

    • Nomes de cores:
      • Limitações: Só existem 147 nomes padrão.
      • Não é preciso decorar, mas é pouco flexível.
      • Uso prático: Bom para testes rápidos, mas raramentes usados em projetos profissionais.
      • Exemplo: red, blue, green, black, white, gray, orange, etc.
    • Códigos hexadecimais:
      • Formato: # seguido por 6 dígitos hexadecimais (0-9, A-F);
      • e amplamente utilizado em projetos profissionais.
      • Exemplo: #FF0000 (vermelho), #0000FF (azul);
    • RGB:
      • Formato: rgb(quantidade de vermelho, quantidade de verde, quantidade de azul);
      • Cada valor varia de 0 a 255;
      • Permite criar uma ampla gama de cores.
      • Exemplo: rgb(255, 0, 0) é vermelho puro, rgb(0, 255, 0) é verde puro.
    • RGBA:
      • Semelhante ao RGB, mas com um quarto valor para opacidade (0 a 1);
      • Permite definir a transparência da cor;
      • Exemplo: rgba(255, 0, 0, 0.5) é vermelho com 50% de opacidade.
    • HSL:
      • Formato: hsl(hue, saturation, lightness);
      • Hue (matiz) varia de 0 a 360 graus (círculo de cores);
      • Saturation (saturação) e lightness (luminosidade) variam de 0% a 100%;
      • Exemplo: hsl(0, 100%, 50%) é vermelho puro, hsl(240, 100%, 50%) é azul puro.
    • HSLA:
      • Semelhante ao HSL, mas com um quarto valor para opacidade (0 a 1);
      • Permite definir a transparência da cor;
      • Exemplo: hsla(0, 100%, 50%, 0.5) é vermelho com 50% de opacidade.

    Exemplo de uso de cores no CSS:

        body {
        background-color: #f0f0f0;
    }
    h1 {
        color: rgb(255, 0, 0);
    }
    p {
        color: rgba(0, 0, 255, 0.5);
    }

    Box Model

    O modelo de caixa do CSS (Box Model) é um conceito fundamental que define como os elementos HTML ocupam espaço na página. Todo elemento renderizado em um site é, na prática, uma "caixa" composta por quatro camadas:

    1. Content — o conteúdo propriamente dito (texto, imagem, etc.);
    2. Padding — o espaço interno entre o conteúdo e a borda da caixa;
    3. Border — a borda da caixa (visível ou não);
    4. Margin — o espaço externo entre a caixa e outros elementos.

    Essas quatro partes formam a estrutura visual e o comportamento do elemento em relação ao layout.

    A estrutura de um elemento no modelo de caixa (Box Model) é a seguinte:

     
╔════════════════════════════════════════╗  ← margin
║  ┌──────────────────────────────────┐  ║
║  │          padding                 │  ║
║  │  ┌────────────────────────────┐  │  ║
║  │  │         content            │  │  ║
║  │  └────────────────────────────┘  │  ║
║  └──────────────────────────────────┘  ║
╚════════════════════════════════════════╝  ← border

    O padding afasta o conteúdo da borda interna. Já a margin afasta o elemento de outros elementos na página. A border pode ter cor, espessura e estilo, ou pode nem aparecer — mas ela sempre existe logicamente.

    E para estilizar esses elementos, usamos propriedades CSS como no exemplo abaixo:

    .caixa {
  width: 200px;
  padding: 20px;
  border: 2px solid black;
  margin: 30px;
}

    Nesse exemplo, o elemento terá:

    • 200px de largura só para o conteúdo;
    • 20px de espaço interno (padding);
    • Uma borda de 2px ao redor da caixa;
    • 30px de distância dos elementos externos (margin).

    JavaScript ✶⋆.˚

    Introdução ao JS

    O que é JavaScript?

    JavaScript é uma linguagem de programação interpretada e orientada a objetos, utilizada principalmente no desenvolvimento web para tornar as páginas interativas e dinâmicas. Junto ao HTML e ao CSS, compõe a base da construção de interfaces na web.

    Foi criada originalmente para rodar no navegador e permitir que páginas HTML respondessem a ações do usuário, como cliques, preenchimento de formulários, animações e mudanças de conteúdo em tempo real.

    Sem o JavaScript, uma página é apenas estática – ou seja, só exibe conteúdo, mas não reage a nada.

    Para que serve?

    JavaScript é usado para adicionar comportamento às páginas. Com ele, é possível:

    • Manipular elementos HTML e alterar seu conteúdo, aparência ou estrutura (DOM);
    • Responder a eventos do usuário (como clique, rolagem ou digitação);
    • Validar formulários antes do envio ao servidor;
    • Controlar o fluxo do programa com condições e repetições;
    • Armazenar dados temporariamente no navegador (ex: localStorage);
    • Fazer requisições a servidores sem recarregar a página (AJAX);
    • Criar animações e efeitos dinâmicos.

    Como o Js funciona no navegador

    Onde o Js é executado?

    No contexto da web, o JavaScript é executado diretamente pelo navegador do usuário (como Chrome, Firefox ou Edge). Esses navegadores possuem mecanismos internos — chamados *motores JavaScript* (como o V8, no Chrome) — que interpretam o código em tempo real e o executam.

    Isso significa que o JS não precisa de instalação nem compilação prévia para funcionar: basta um navegador. Ao abrir uma página que contenha JavaScript, o navegador processa o código e o executa automaticamente.

    Hoje, o JavaScript também pode ser executado fora do navegador (como no Node.js), mas seu uso mais comum ainda é no front-end.

    Carregamento e execução

    JavaScript funciona interpretando e executando código diretamente no navegador do usuário. Ele pode ser incluído em um documento HTML de três maneiras principais:

    1. Inline: Adicionando código JavaScript diretamente na tag HTML usando o atributo onclick, por exemplo.
      2.     <button onclick="alert('Olá!')">Clique aqui</button>
    2. Internal: Incluindo um bloco de script dentro da tag <head> ou <body> do HTML.
      3.     <head>
        <script>
            function mostrarAlerta() {
                alert('Olá!');
            }
        </script>
    </head>
    3. External: Criando um arquivo JavaScript separado e vinculando-o ao HTML.
      4.     <head>
        <script src="script.js"></script>
    </head>

    A forma mais recomendada é a externa, pois permite separar o conteúdo (HTML) da lógica (JavaScript), facilitando a manutenção e reutilização do código.

    Interação com HTML e CSS

    O JavaScript se conecta ao HTML e ao CSS por meio da DOM (Document Object Model), uma representação em árvore da estrutura da página.

    Com ele, você pode:

    • Selecionar elementos da página (`document.querySelector`, `getElementById`, etc.);
    • Alterar conteúdo e atributos (`element.innerText`, `element.setAttribute`);
    • Modificar estilos diretamente (`element.style.color = "red"`);
    • Adicionar ou remover classes CSS;
    • Inserir, mover ou remover elementos da página.

    Variáveis e tipos de dados

    Em JavaScript, variáveis são usadas para armazenar valores e podem ser declaradas usando as palavras-chave var, let ou const. Cada uma tem suas particularidades:

    • var: Declara uma variável com escopo global ou de função. Pode ser reatribuída e redeclarada.
    • let: Declara uma variável com escopo de bloco. Pode ser reatribuída, mas não redeclarada no mesmo escopo.
    • const: Declara uma constante, cujo valor não pode ser reatribuído. Também tem escopo de bloco.

    Exemplo de declaração de variáveis:

        var nome = 'João';
    let idade = 30;
    const pi = 3.14;

    Os tipos de dados mais comuns em JavaScript incluem

    • String: Texto, delimitado por aspas simples ou duplas (ex: 'Olá', "Mundo");
    • Number: Números inteiros ou decimais (ex: 42, 3.14);
    • Boolean: Valores lógicos verdadeiro (true) ou falso (false);
    • Array: Lista de valores, delimitada por colchetes (ex: [1, 2, 3], ['maçã', 'banana']);
    • Object: Estrutura de chave-valor, delimitada por chaves (ex: { nome: 'João', idade: 30 });
    • Null: Valor nulo, representado por null;
    • Undefined: Valor indefinido, representado por undefined.

    Operadores

    Operadores Aritméticos

    Usados para realizar operações matemáticas:

    • + (adição)
    • - (subtração)
    • * (multiplicação)
    • / (divisão)
    • % (módulo, resto da divisão)

    Operadores de Comparação

    Usados para comparar valores e retornar verdadeiro (true) ou falso (false):

    • == (igualdade)
    • != (desigualdade)
    • === (igualdade estrita, compara valor e tipo)
    • !== (desigualdade estrita, compara valor e tipo)
    • > (maior que)
    • < (menor que)
    • >= (maior ou igual a)
    • <= (menor ou igual a)

    Operadores Lógicos

    Usados para combinar expressões booleanas:

    • && (E lógico, retorna true se ambas as expressões forem verdadeiras)
    • || (OU lógico, retorna true se pelo menos uma expressão for verdadeira)
    • ! (NÃO lógico, inverte o valor booleano)

    Estrutura de controle

    Estruturas de controle são usadas para controlar o fluxo de execução do código, permitindo tomar decisões e repetir ações. As principais estruturas de controle em JavaScript são:

    • if...else: Condicional que executa um bloco de código se uma condição for verdadeira, e outro bloco se for falsa.
    • switch: Estrutura condicional que compara uma expressão com vários casos possíveis.
    • for: Laço de repetição que itera sobre um bloco de código um número específico de vezes.
    • while: Laço de repetição que executa um bloco de código enquanto uma condição for verdadeira.
    • do...while: Semelhante ao while, mas garante que o bloco seja executado pelo menos uma vez antes de verificar a condição.
